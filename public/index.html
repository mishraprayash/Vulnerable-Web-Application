Sentinel: Vulnerable Web Application

A deliberately vulnerable application for cybersecurity education
⚠️ WARNING: This application contains intentional security vulnerabilities for educational purposes only. DO NOT deploy in production environments or on publicly accessible servers.

How to Use This Application

Each vulnerability has its own dedicated page where you can experiment with different attack payloads. Click on any of the vulnerabilities below to start exploring. Use this application to understand how these vulnerabilities work and how to protect against them in real applications.

SQL Injection (SQLi)

Exploiting SQL databases by injecting malicious SQL code into queries. Try inserting special characters and SQL statements to manipulate the database query.

Try SQLi Demo

NoSQL Injection

Similar to SQL injection but targeting NoSQL databases like MongoDB. Try using JSON objects with operators to bypass authentication.

Try NoSQL Injection

Server-Side Request Forgery (SSRF)

Forcing the server to make unintended requests to internal or external systems. Manipulate the target URL to access restricted resources.

Try SSRF Demo

Command Injection

Executing arbitrary system commands through vulnerable application functions. Try appending commands using operators like ; | && to run multiple commands.

Try Command Injection

Cross-Site Scripting (XSS)

Injecting malicious scripts that execute in users' browsers. Try inserting HTML tags and JavaScript code to see how they execute in the page context.

Try XSS Demo

Local File Inclusion (LFI)

Accessing files on the server through path manipulation. Try accessing system files like /etc/passwd or application files outside the intended directory.

Try LFI Demo
